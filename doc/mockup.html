Design Mockup for Views

This document contains a mockup of each view that I consider necessary for Spoodle, except for the views that consist of just a form.
Colours are not final!

Welcome view

This view greats all visitors that are not logged in. It contains a carousel with some fancy images and a list of features that Spoodle offers. The aim is to get visitors to click on the link that takes them to the register form.

Home view

This view is default page for logged in users. It contains a list of all events, split into separate tabs for each possible state an event can have. Only one tab is shown at a time, the others are hidden.

Event views

Invitation

Inivitations are events that the user has been inivted to but where he has NOT YET selected his available dates.

Upcoming, variable Date

The difference between this and the invitation is that the user has already selected his available dates but the date of the event itself has not yet been finalized.

Upcoming, fixed Date

Here there is afinal date when the event will take place (but it's in the future so it isn't passed yet).

Passed

This is the view for events that have passed. The organizer of an event has the ability to add some data (like duration, distance, GPS positions, ...) to the database and visualise this data in the passed event view.

Friends view

This view is where all friends of a user are listed and where he can reach a link to search for users (by name, email, ...) to add more friends.

Statistics view

This view displays aggregated statistics (per day, week, month, year) about data (like distance, duration, ...) of all the events the user has taken part.
